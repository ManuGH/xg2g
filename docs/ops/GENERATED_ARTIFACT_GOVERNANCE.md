# Generated Artifact Governance

Canonical governance contract for repo-owned generated truth in `xg2g`.

This document answers four questions for each generated artifact class:

1. Which files are under governance.
2. Which command is the canonical generator.
3. Which source files are the truth inputs.
4. Whether drift is a blocker and whether outputs must be committed.

## Policy

- Generated artifacts under this contract are `committed required`.
- Drift in committed generated artifacts is a merge blocker.
- Generators may be run during verification, but verification must fail instead of silently repairing drift.
- `ungoverned generated truth` is a blocker:
  any tracked repo file with a repo-owned generated marker, or any tracked file under a governed generated prefix, must belong to a documented artifact class below.

## Local-only Generated Outputs

These outputs are intentionally not governed as committed repo truth:

| Path / Class | Canonical Producer | Commit Policy | Drift Policy |
| --- | --- | --- | --- |
| `frontend/webui/dist/**` | `make ui-build` | never committed | tracked presence is a blocker |
| `artifacts/**`, `coverage.out`, `coverage.html` | tests / coverage targets | never committed | tracked presence is a blocker |

## Verification

The governance entrypoint is `backend/scripts/verify-generated-artifacts-contract.sh`.

The full generated-artifact verification path is:

1. `make verify-generated-artifacts`
2. `make ci-pr`

## Detection Rule

`ungoverned generated truth` includes:

- tracked files with repo-owned generated markers such as:
  - `GENERATED FILE - DO NOT EDIT`
  - `Code generated by github.com/oapi-codegen/oapi-codegen`
  - `This file is auto-generated by @hey-api/openapi-ts`
  - `MECHANICALLY GENERATED by backend/scripts/generate-consumption-types.mjs`
  - configgen-generated markers and generated config blocks
- tracked files under committed generated prefixes such as:
  - `frontend/webui/src/client-ts/`
  - `backend/internal/control/http/dist/`

If such a file is not covered by one of the governed classes above, verification must fail.
